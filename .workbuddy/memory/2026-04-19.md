# 2026-04-19 工作日志

## 分析任务：印尼市场风控智能体功能增强方案

### 项目背景
- 基于 yu-ai-agent（程序员鱼皮教学项目），Spring Boot 3.4.4 + Spring AI 1.0.0 + Vue 3
- 用户希望将该 AI 智能体改造为面向印度尼西亚市场的风控智能体

### 项目现有核心能力
- ReAct Agent（YuManus）：Think-Act 循环，最多 20 步，SSE 推送
- 工具体系 7 个：联网搜索、抓取、文件、下载、终端、PDF、终止
- RAG 知识库：PgVector + SimpleVectorStore + 查询重写
- Advisor 链：可插拔拦截器

### 主要功能增强方案（8 大模块）
1. 竞品功能分析智能体（CompetitorAnalysisAgent）
2. 多头借贷检测（MultiLoanDetectionTool + AFPI 数据接入）
3. 反欺诈加强维护（FraudDetectionTool：KTP/人脸/设备指纹）
4. 风控关键指标异常提醒（RiskMetricsMonitorAgent + WhatsApp 告警）
5. OJK 合规功能（POJK 40/2024，SLIK 征信接入）
6. 印尼本地化 RAG 知识库（OJK 法规、欺诈模式）
7. 多智能体协作架构（Orchestrator + 5 子 Agent）
8. 伊斯兰金融合规（Sharia Finance）

### 印尼特有风险点
- 多头借贷是最大信用风险
- KTP（身份证）伪造欺诈普遍
- 无征信人群约 5500 万（替代数据评分需求大）
- 区域信用文化差异（17,000 个岛屿）
- 节假日风险（开斋节前后逾期率高）

### 三方数据源规划
- SLIK（OJK 官方征信，需牌照）
- Pefindo / Experian ID（替代征信）
- 电信数据（Telkomsel/XL/Indosat）
- ADVANCE.AI Guardian（设备指纹）
