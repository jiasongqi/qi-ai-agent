# 2026-04-23 工作记录

## 全项目代码 Review（非竞品模块）

### 高危安全问题
1. `TerminalOperationTool`：无白名单的 RCE，可执行任意系统命令，已注册到 YuManus 对外接口
2. `FileOperationTool` / `ResourceDownloadTool` / `PDFGenerationTool`：fileName 未做路径规范化，存在路径遍历漏洞
3. `ResourceDownloadTool`：无限制 URL 下载，有 SSRF 风险，无文件大小限制
4. `WebScrapingTool`：返回完整 HTML（token 浪费），无超时，有 SSRF

### Bug
- `FileBasedChatMemory`：Kryo 是 static 单例，线程不安全，并发读写会崩溃（需改为 ThreadLocal）
- `WebSearchTool`：`organic_results` 为 null 时 NPE
- `ToolCallAgent.think()` 异常后不置 ERROR 状态，Agent 继续无效循环
- `YuManus` 标了 @Component 但 Controller 里 new 新实例，单例状态污染风险
- `LoveApp.doChatWithRag()`：每次 new MyLoggerAdvisor，且和 defaultAdvisors 重复注册

### 配置/规范
- CorsConfig 全域 allowCredentials(true) + allowedOriginPatterns("*")，CSRF 风险
- application.yml 数据库密码和内网 IP 明文
- Manus 接口无鉴权但有命令执行能力
- maxTokens=2000 对分析类任务太小
- WebSearchTool @Tool description 写成 Baidu 但实际走 SearchAPI

## CompetitorAnalysisAgent 代码 Review

对竞品分析智能体（`src/main/java/com/yupi/yuaiagent/competitor/`）进行了全面代码审查，发现以下关键问题：

### 高优先级缺口（断链问题）
1. `executeDailyMonitor()` 只采集评论，没有触发 LLM 分析，`CompetitorAnalysisAgent` 从未被调用
2. `triggerManualAnalysis()` 只调 `fetchAppReviews`，没有执行分析
3. 报告生成后没有持久化到 `t_competitor_analysis_report`
4. `OfficialSiteMonitor.monitorPolicyChange()` 从未被任何流程调用

### 中优先级问题
- Prompt 中只显示 App ID，没有 App 名称
- `CompetitorComparisonTool` 注册了 `@Tool` 但 ChatClient 调用时没有 `.tools()` 注册，LLM 无法 Function Calling
- 差评主题关键词硬编码且覆盖不足
- `generateDiff()` 对齐式 diff 有噪音 Bug，建议用 java-diff-utils
- `sentimentScore` 和 `keywords` 字段永远为 null

### 工程规范问题
- Agent 是 `@Component` 但有实例状态变量，并发不安全
- Controller 直接注入 Mapper，绕过 Service 层
- 预警生成逻辑缺失（AlertService 实现了但从未被调用）
- 缺少 Google Play 评论爬虫（印尼 Android 用户占~85%）
- `AppMonitoringConfig` 缺少 `official_url` 和 `policy_url` 字段
